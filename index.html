GitHub Commands Explained

Made by: Adrián Jiménez Llorens and Alejandro Marco Sánchez

Commit

The `commit` command saves your changes to the repository's timeline, creating a snapshot of your work. Each commit has a unique ID and a message describing the changes.

Made by: Adrián Jiménez Llorens

Branch

The `branch` command allows you to create a separate working copy of your project. This is useful for testing features without affecting the main project history.

Made by: Alejandro Marco Sánchez

Checkout

The `checkout` command lets you switch between branches or specific versions in your project. It's like jumping to a specific chapter in your project's history.

Made by: Adrián Jiménez Llorens

Cherry-pick

With `cherry-pick`, you can apply a specific commit from one branch to another without merging the entire branch.

Made by: Alejandro Marco Sánchez

Reset

The `reset` command undoes recent commits by moving the branch pointer back to a previous state. Be careful, as this can delete changes!

Made by: Adrián Jiménez Llorens

Revert

The `revert` command creates a new commit that undoes the changes made by a previous commit. It does not delete any history.

Made by: Alejandro Marco Sánchez

Rebase

The `rebase` command restructures the project history, making it appear linear by applying changes from one branch onto another.

Made by: Adrián Jiménez Llorens

Merge

With `merge`, you combine changes from one branch into another. This command preserves the history of both branches.

Made by: Alejandro Marco Sánchez

Clone

The `clone` command creates a copy of a repository on your local machine. It’s used to start working on an existing project.

Made by: Adrián Jiménez Llorens

Pull

The `pull` command fetches changes from a remote repository and merges them into your local branch.

Made by: Alejandro Marco Sánchez

Push

The `push` command uploads your local changes to the remote repository, making them accessible to others.

Made by: Adrián Jiménez Llorens

Fetch

The `fetch` command downloads changes from the remote repository without merging them into your local branch.

Made by: Alejandro Marco Sánchez

Stash

The `stash` command temporarily saves your changes without committing them, allowing you to work on something else.

Made by: Adrián Jiménez Llorens

Log

The `log` command shows the commit history, providing details about each change, including the author and commit message.

Made by: Alejandro Marco Sánchez

Diff

The `diff` command compares changes between files, branches, or commits, highlighting additions and deletions.

Made by: Adrián Jiménez Llorens

Tag

The `tag` command marks specific commits as important milestones, like software releases.

Made by: Alejandro Marco Sánchez

Status

The `status` command displays the current state of your repository, including modified, staged, and untracked files.

Made by: Adrián Jiménez Llorens

Remote

The `remote` command manages connections to remote repositories, allowing you to add or remove them.

Made by: Alejandro Marco Sánchez

Blame

The `blame` command shows who last modified each line of a file, helping you identify contributors to specific changes.

Made by: Adrián Jiménez Llorens

Apply

The `apply` command applies changes stored in a patch file to your repository.

Made by: Alejandro Marco Sánchez

Amend

The `amend` command modifies the last commit, allowing you to correct mistakes or add files you forgot to include.

Made by: Adrián Jiménez Llorens

Clean

The `clean` command removes untracked files and directories from your working directory.

Made by: Alejandro Marco Sánchez

Squash

The `squash` command combines multiple commits into a single one, simplifying the project's history.

Made by: Adrián Jiménez Llorens
